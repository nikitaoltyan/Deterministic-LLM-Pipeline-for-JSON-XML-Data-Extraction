# Цель проекта: детерминированный конвейер структурирования данных

## Общая идея

Цель проекта — создать архитектуру и программную систему, которая преобразует неструктурированные текстовые данные в строго валидные структурированные представления (JSON или XML), используя большие языковые модели (LLM), но при этом устраняет их стохастическую природу и обеспечивает воспроизводимость результата.

Система должна реализовывать **детерминированный конвейер обработки**, в котором LLM используется только как один из компонентов, а итоговый результат контролируется формальными ограничениями.

Главная идея — объединить:

- вероятностную генерацию (LLM)
- формальные методы (грамматики, схемы, типизация)
- детерминированные процедуры валидации и восстановления

в **единый управляемый процесс преобразования данных**.

---

## Формальная постановка задачи

Задача рассматривается как отображение:

T → S → Y

где:

T — исходный текст  
S — структурированная форма (JSON/XML)  
Y — типизированный объект прикладной модели данных

Полный процесс представляет собой композицию функций:

φ_pipeline = ψτ ∘ κ ∘ ρ ∘ νΣ ∘ γG ∘ μθ ∘ π

где:

π — предобработка текста  
μθ — генерация структуры LLM  
γG — грамматически ограниченное декодирование  
νΣ — проверка по схеме  
ρ — восстановление структуры  
κ — канонизация  
ψτ — типизация

---

## Ключевая цель архитектуры

Обеспечить выполнение четырех инвариантов корректности:

### I1 — синтаксическая валидность
Результат всегда является корректным JSON или XML.

### I2 — соответствие схеме
Структура удовлетворяет JSON Schema или XSD.

### I3 — строгая типизация
Все значения приводимы к заданной системе типов.

### I4 — детерминизм
При одинаковом входе и конфигурации среды результат всегда одинаковый.

---

## Основная архитектурная идея

Вся система строится как **детерминированный pipeline**, состоящий из независимых модулей:

Preprocessor  
→ Prompt Builder  
→ LLM Core  
→ Grammar-Constrained Decoder  
→ Validator  
→ Repair & Retry  
→ Canonicalizer  
→ Typifier  
→ Logger & Trace

Каждый модуль:

- выполняет одну формальную функцию
- имеет фиксированные входы/выходы
- является детерминированным

---

## Роль LLM в системе

LLM **не определяет результат напрямую**.

Она используется только для генерации **кандидатной структуры**, которая затем проходит последовательность формальных ограничителей.

Все гарантии корректности обеспечиваются **не моделью, а архитектурой конвейера**.

---

## Итог

Проект направлен на создание **универсального архитектурного шаблона структурирования данных**, который:

- совместим с любыми LLM
- обеспечивает строгую валидность данных
- воспроизводим
- пригоден для промышленного использования

Такая архитектура превращает вероятностную генерацию текста в **формально управляемый процесс структурирования данных**.
